Teaching agents to reason like IPL analysts.

IPL Scorecard Reasoning is a sports analytics environment built around real Cricsheet Indian Premier League data. Agents do not just read an answer from a JSON blob. They study solved examples, infer a scoring rule, and apply it to a fresh scorecard.

Environment Context

Why cricket scorecards?

Cricket is full of compact symbolic data: innings totals, wickets, over-by-over scoring, chase targets, and contextual rules. This makes it a good hackathon benchmark because the answer is deterministic, but the agent still needs to learn the right operation from context.

The environment uses a compact derived dataset from Cricsheet IPL matches. Result fields and precomputed helper totals are removed from the challenge view, forcing the model to work from the raw scorecard.

Dataset

Real IPL Data

Ball-by-ball Cricsheet data is transformed into scorecard summaries: teams, venues, innings totals, wickets, over runs, top batters, and bowling figures.

Learning Signal

Two Examples, One Test

Every episode gives two solved examples for one hidden pattern, then asks the agent to apply that same pattern to a held-out IPL match.

Rules Tested

Cricket Arithmetic

Tasks cover powerplay comparisons, death-over totals, chase requirements, and winner-margin inference from innings scorecards.

Reward

Scalar Credit

Full answers receive 1.0. Partially correct answers, such as the right team and number with incomplete phrasing, receive partial credit.

Build Process

From raw cricket data to agent evaluation

01 Collect

Pull public IPL match data from Cricsheet and derive compact scorecards.

02 Hide

Strip answer-leaking result fields and precomputed aggregates from challenge observations.

03 Teach

Show two solved examples so the model can infer the calculation pattern in-context.

04 Evaluate

Grade the held-out answer with scalar reward and replay the trace in Mesocosm.
